## 1. 评测集怎么来（`data/rag_eval_build.py`，免标注）
**反向生成**：从知识库挑已知父块 → LLM 出「只能用这段回答的问题+标准答案」→ **那个父块天生就是 ground-truth 依据**，不用人工标。
- 分层抽样避免口碑淹没：口碑/乘联会/研报 各抽若干。
- 难度梯度（§12.2）：**single**(单段 38) / **cross**(跨段需综合 6) / **negative**(库里没有 10，验防幻觉)。
- 产物 `data/rag_eval/eval_set.jsonl`：`{qid,type,source,question,ground_truth,gt_doc_id,gt_parent_ids}` 共 54 条。

## 2. 指标怎么算（`data/rag_eval_run.py`，检索/生成分开）
| 指标 | 测什么 | 算法 |
|---|---|---|
| context_recall | gt 依据召回率 | gt 父块是否在重排后 Top-K 子块的父块里 |
| context_precision | 召回相关占比 | Top-K 父块中属 gt 的占比 |
| MRR | 首个命中排名 | 首个 gt 父块名次的倒数 |
| faithfulness | 答案是否忠于召回、不编造 | LLM-as-judge（是/否）|
| answer_correctness | 是否覆盖标准答案要点 | LLM-as-judge（正确/部分/错误）|
| abstain_rate | 库里没有时是否正确拒答 | negative 题 has_answer 应为 false |

## 3. 基线结果（2026-05-25，54 题）

```
【检索】 context_recall=0.455  context_precision=0.081  MRR=0.388
        单段=0.500 跨段=0.167 ｜ 按来源: 口碑=0.750 乘联会=0.278 研报=0.300
【生成】 faithfulness=0.341  answer_correctness=0.420
【防幻觉】 abstain_rate=1.000
```

## 4. 怎么读这些数（诚实解读）
- ✅ **abstain=1.000**：10 条「库里没有」全部正确拒答，**零幻觉**——防幻觉机制（召回空/低分→不答）有效。
- ✅ **口碑 recall=0.750**：占主体的语义型语料，检索是好的。
- ⚠️ **乘联会/研报 recall 仅 0.278/0.300**：政策/数字/专名类事实**明显召回不足**。
- ⚠️ **context_precision=0.081 是指标特性，不是「8% 准」**：单条只有 1 个 gt 父块，却召回 ~8 个子块/多个父块，precision 上限本就 ~0.12；这里看 **recall + MRR** 更有意义。
- ⚠️ **faithfulness/correctness 偏低（0.34/0.42）被检索失败拖累**：检索没召回到→无答案或答错→这两项记 0。根子在检索，不在生成。

## 5. 定位到的真问题（评测台立刻就赚回来了）
**根因（已验证）**：混合召回的「关键词/全文」这一路用 PG `to_tsvector('simple')`，**对中文不分词**：
```
全文召回「疲劳驾驶新规认定标准」→ 0 条
全文召回「广东智能网联汽车」    → 0 条
全文召回「800V」               → 6 条   ← 只有纯英文/数字 token 有效
```
→ 中文 query 下混合召回**退化成纯向量**；而政策/数字/专名正是向量的弱项（PRD §5.4 要 BM25 补的就是这个）——补救通道对中文是失效的，于是 cpca/研报这类 niche 事实召回垮掉。

**这正是评测的价值**：一句「召回不准」被定位成「中文全文检索通道失效」这个可操作的具体缺陷。

## 5.1 ✅ 已修复 + 回归验证（闭环）
**修法（纯应用层，不依赖 pg_jieba 扩展）**：入库用 **jieba** 把 content 切成空格分隔词存进新列 `content_tokens`，
PG `simple` 对空格即能分词；查询也 jieba 切词、用 `|`(OR) 组 tsquery 提召回。
代码：`app/rag/text.py`(分词) + `app/rag/pg.py`(keyword_search 改查 content_tokens)；
存量 28629 chunk 用 `data/rag_fts_backfill.py` 回填分词 + 重建 GIN 索引（无需重算向量）。

**回归对比（同一 54 题评测集，修复前 → 修复后）**：

| 指标 | 修复前 | 修复后 |
|---|---|---|
| context_recall | 0.455 | **0.830** |
| MRR | 0.388 | **0.759** |
| recall 乘联会 / 研报 / 跨段 | 0.28 / 0.30 / 0.17 | **0.89 / 0.85 / 0.92** |
| faithfulness | 0.341 | **0.705** |
| answer_correctness | 0.420 | **0.795** |
| abstain_rate（防幻觉） | 1.000 | 1.000 |

> 简历叙事：「评测定位中文全文检索失效 → jieba 应用层分词修复 → **context_recall 0.46→0.83、答案正确率 0.42→0.80**，零幻觉保持」。
> 检索修好后生成两项（faithfulness/correctness）自然回升——印证「生成差是被检索拖累」的判断。

## 6. 据基线的改进路线（按指标定位，§12.2.4）
| 现象 | 指标 | 修法 |
|---|---|---|
| 中文关键词召回 0 | recall(cpca/pdf) 低 | **上中文分词全文检索 pg_jieba/zhparser**，或入库时 jieba 预分词写进 tsvector（首选，立竿见影）|
| 跨段 recall 0.167 | recall_cross 低 | 放大 Top-K、父块归并保更多相邻块 |
| 生成两项偏低 | faithfulness/correctness | 检索修好后自然回升；再不行收紧 prompt |
- 每改一版回归本评测集，记录提升（简历叙事：「补中文 BM25 使乘联会类 recall 从 0.28→X」）。
