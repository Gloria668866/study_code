# 01 · 产品方向

## 是什么
**车市镜（EV-MarketLens）**：面向新能源汽车行业的**对话式市场情报 Agent**。
用户用大白话提问，系统自动判断意图：
- 要数据 → **Text2SQL** 查结构化库 → 自动出**图表 + 结论 + 归因**；
- 要解读 → **RAG** 查用户上传的**知识库文档**（行研报告/政策/车评）→ 给**带引用**的答案；
- 两者都要 → Agent 编排并行调两个脑再合并。

一句话：把「找数据、做分析、读报告」合并到一个对话框。

## 为什么做（目标）
- 作品级**真实产品**，主要服务 **2026 秋招拿高薪**，兼顾未来创业（秋招窗口约 2026-08，本记忆写于 2026-05，剩约 3 个月）。
- 岗位定位：**大模型应用 / Agent 工程**（当前最贵最缺），所以核心卖点压在 Agent/LLM 能力，**不是**数据可视化。

## 为什么是「新能源车市情报」这个方向
- 避开「电商数据可视化」烂大街、面试撞车的问题。
- 新能源是热门招聘赛道（车企、科技大厂都在招），公开数据可爬。
- 同样一份数据，定位成「市场/竞品情报」比「电商分析」高一档。

## 核心差异化（区别于 99% 简历项目）
1. **是 Agent 不是看板**：LLM 规划→Text2SQL→自校验重试→图表→归因。
2. **双脑**：结构化(Text2SQL) + 非结构化(RAG)，还用 Agent 编排路由——绝大多数项目只占其一。
3. **通用引擎 + 可插拔领域包**：引擎 schema-agnostic，新能源车市是第一个领域包，「换行业=换一个目录」。进可攻（讲平台）退可守（有垂直落地 demo）。

## 被否决/调整过的方向（避免重走）
- ❌ 纯汽车经销商内部数据（销售/售后）：内网涉密、爬不到。
- ❌ 纯「通用任意数据平台」：3 个月会做得宽而浅、面试被问穿。
- ❌ 直接叫「电商分析」：撞车、显廉价。
- ✅ 最终：垂直落地（新能源车市）+ 通用底座叙事。
