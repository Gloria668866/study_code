## 主数据源（已锁定）：懂车帝销量榜 API
- 接口：`GET https://www.dongchedi.com/motor/pc/car/rank_data`
- 参数：`rank_data_type=11`(销量榜)、`new_energy_type=0全部/1纯电/2插混/3增程`、`month=`(空=最新，可填 YYYYMM 拉历史)、`count`/`offset`(翻页)、`brand_id`/`price`(筛选)
- 请求头：带 `Referer: https://www.dongchedi.com/sales` 即可，**免登录、免签名**，返回干净 JSON
- 一个接口即含：品牌、车系、销量(count)、排名(rank + last_rank)、指导价(min/max_price)、经销商价(dealer_price)、口碑数(car_review_count)

## 数据量（实测，2026-05-22）
- 单月新能源在榜 ≈ 199 车系；4 种能源类型都可拉。
- 历史抽样 2024-01 ~ 2025-04 全部 OK（≥16 个月可回溯）。
- 估算：新能源 × 16+ 月 ≈ **近万行销量事实**；用户确认「够了」。

## 第二数据源（已补，2026-05-25）：懂车帝口碑页（补三字段 + RAG 语料）
- 销量接口给不了的三字段，从**口碑/评分页**补：`GET https://www.dongchedi.com/auto/series/score/{series_id}`
- 懂车帝是 **Next.js SSR**：候选 JSON 接口全 404，但页面 HTML 里 `<script id="__NEXT_DATA__">` 含整页数据，
  解析 `props.pageProps` 一次拿全（免登录、带 Referer 即可）：
  - 评分 `seriesHomeHead.total_score`（**值×100**，404=4.04 分）+ 7 个分项
  - 级别 `seriesHomeHead.car_type`（如「中型SUV」）
  - 续航/动力 `seriesHomeHead.pc_config.recharge_mileage`（593-821km）/`total_electric_power`/...
  - 评论原文 `reviewListData.review_list[].content`（首页约 14 条/车系，作 RAG 语料）
- 脚本 `data/crawl_koubei.py`（沿用 crawl_sales 范式，按车系遍历 dim_series 的 409 个 id，幂等）。
- 实测：409 车系全采、0 空；detail=409、reviews=5280；字段清单见 `data/probe/口碑车系详情字段清单.md`。

## 已定死的表结构 → `sql/schema.sql`
基于**实测真实字段**用 Kimball 维度建模定稿（方法论写在 PRD-1 §7.1）。

**维度表**
- `dim_brand`：brand_id(PK), brand_name, sub_brand_id, sub_brand_name, country_type(enrich), is_new_force(enrich)
- `dim_series`：series_id(PK), series_name, brand_id(FK), segment(级别,口碑页补), powertrain(能源类型推断), endurance_km(续航上限,口碑页补), guide_price_min/max, image_url
- `dim_date`：date_id(YYYYMM,PK), year, month, quarter, ym

**事实表**
- `fact_sales_rank`（车系×月×能源类型）：series_id, date_id, new_energy_type, rank_type, rank, last_rank, **volume(销量)**, source/source_url/crawl_time/etl_version, UNIQUE(series_id,date_id,new_energy_type,rank_type)
- `fact_price`（车系×快照日）：series_id, date_id, guide_price_min/max, price_text, dealer_price_text, has_dealer_price, descender_price, +血缘
- `fact_review`（车系×快照日）：series_id, date_id, review_count, score(nullable), sentiment(nullable), +血缘

**应用层表** → `sql/schema_app.sql`（PostgreSQL，向量需 pgvector）：
- `users`（登录模块：username/password_hash(bcrypt)/nickname/...）
- `conversation` / `message`（会话历史，按 user_id 归属）
- `kb_document` / `kb_chunk`（知识库，kb_chunk.embedding=vector(1024) BGE）
- 设计依据 PRD-2 §6/§17。**登录模块是后补的**：之前各表已带 user_id，但缺 users 表与登录流程，已补 schema_app.sql + 派给前后端实现（用户名+密码+JWT，轻量版）。
