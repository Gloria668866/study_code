# 工业级 Embedding 模型选型与评估思路

## 1. 选型目标
嵌入模型是将文本转换为向量表示的核心组件，在 RAG、搜索、推荐系统中关键。选型目标：高准确率、低延迟、低成本、兼容领域（如中文多语言支持）。

## 2. 评估维度
- **准确率**：检索 hit rate@1/5 (正确上下文在 top-k 中比例), MRR (平均倒数排名), NDCG (归一化折扣累积增益)。使用 MTEB leaderboard 查看通用性能。
- **速度**：嵌入生成时间、检索时间（使用Milvu或其他向量库 测试）。
- **维度**：低维 (384) 快，高维 (1024+) 准确但慢。
- **上下文长度**：长上下文 (8192+ token) 适合长文档 RAG。
- **多语言/领域支持**：中文用 bge-m3/Qwen，代码用 voyage-code-2。
- **成本**：开源免费 (Hugging Face), API 按调用收费 (OpenAI ~$0.0001/1k token)。
- **部署**：开源支持本地/私有，API 易用但依赖云。

## 3. 常见模型推荐
- **开源**：bge-m3 (多语言, 1024 dim), e5-large (多任务, 1024 dim), gte-large (通用, 1024 dim), all-MiniLM-L6-v2 (小模型, 384 dim, 速度快)。
- **商业**：OpenAI text-embedding-3-large (8192 token, 高精度), Cohere embed-v3 (多语言), Google text-embedding-004 (Vertex AI)。
- **中文专用**：BCEmbedding, Qwen 嵌入系列。

## 4. 评估流程
1. **数据准备**：加载rag数据集，包含问题、上下文、答案。过滤 token 长度。
2. **模型加载**：定义模型列表，动态获取元数据 (dim, max_tokens from config.max_position_embeddings)。
3. **生成嵌入**：转换上下文为向量，测量时间。
4. **构建索引**：使用 Milvus或FAISS 存储向量，测量构建时间。
5. **检索测试**：为问题生成查询向量，检索 top-1，测量时间和准确率/MRR。
6. **可视化**：绘制时间/准确率图。
7. **迭代**：测试 3-5 模型，选择最佳（e.g., 准确率 >90%, 时间 <1s/query）。

## 5. 优化技巧
- **混合嵌入**：dense (语义) + sparse (关键词) 提升精度。
- **微调**：用领域数据 fine-tune (LoRA 低成本)。
- **量化**：fp16/INT8 加速 2x。
- **基准测试**：MTEB, BEIR 数据集。
- **监控**：工业中用 MLflow 跟踪指标，A/B 测试上线模型。

## 6. 风险
- 小数据集问题：用 1000+ 样本测试。
- 成本超支：API 模型监控使用量。
- 安全：开源模型避免数据泄漏。
