# 测试管理与缺陷管理

## 测试管理流程

### 测试计划阶段

测试计划是测试活动的纲领性文档，包含以下内容：

1. **测试范围**：明确哪些功能需要测试，哪些不需要
2. **测试策略**：采用的测试类型和方法
3. **资源分配**：人员、环境、工具
4. **时间安排**：各阶段的时间节点和里程碑
5. **风险评估**：识别可能影响测试进度的风险及应对措施
6. **退出标准**：何时可以结束测试（如：所有严重Bug已修复、覆盖率达到80%等）

### 测试设计与执行

- 测试用例编写：基于需求文档和设计文档
- 测试数据准备：构造正常、异常、边界数据
- 测试环境搭建：确保与生产环境一致
- 测试执行：按照优先级执行测试用例
- 结果记录：详细记录测试结果和发现的问题

### 测试报告

测试报告应包含：
- 测试执行概况（计划的用例数、执行的用例数、通过率）
- 缺陷统计（按严重级别、模块分布）
- 质量评估与建议
- 遗留风险说明

## 缺陷生命周期

```
新建（New） -> 确认（Confirmed） -> 分配（Assigned）
    -> 修复中（In Progress） -> 已修复（Fixed）
    -> 验证（Verified） -> 关闭（Closed）
    
    拒绝（Rejected） <- 重复（Duplicate）
    暂缓（Deferred）
    重新打开（Reopened）
```

### 缺陷报告要素

报告Bug时应包含以下信息：
1. **标题**：简洁描述问题现象
2. **严重级别**：Blocker > Critical > Major > Minor > Trivial
3. **优先级**：立即修复 > 高 > 中 > 低
4. **复现步骤**：详细的操作步骤
5. **预期结果 vs 实际结果**
6. **环境信息**：操作系统、浏览器版本、测试环境
7. **附件**：截图、日志、视频

## 测试管理工具

### JIRA
Atlassian JIRA是最流行的项目管理和缺陷跟踪工具之一：

- 创建和管理Issue（Bug、Task、Story）
- 自定义工作流（如：待处理 -> 进行中 -> 已完成）
- Scrum/Kanban面板
- 与Confluence、Bitbucket等集成

### TestLink
开源的测试管理工具，专注于测试用例管理：

- 测试用例的创建、分类和维护
- 测试计划制定和执行跟踪
- 测试报告生成
- 与JIRA等缺陷管理工具集成

## 测试度量指标

| 指标 | 计算公式 | 用途 |
|------|----------|------|
| 用例通过率 | 通过的用例数 / 总执行用例数 | 衡量测试质量 |
| 缺陷密度 | 缺陷数 / 代码行数(千行) | 评估代码质量 |
| 缺陷修复率 | 已修复缺陷数 / 总缺陷数 | 衡量修复效率 |
| 缺陷重新打开率 | 重新打开的缺陷数 / 总缺陷数 | 衡量修复质量 |
| 测试覆盖率 | 已测试的需求数 / 总需求数 | 衡量测试完备性 |
