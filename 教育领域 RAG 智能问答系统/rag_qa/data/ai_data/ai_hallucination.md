# 模型幻觉问题

## 什么是模型幻觉

模型幻觉（Hallucination）是指大语言模型生成了看似合理但实际不正确或无根据的内容。这是LLM的核心挑战之一，幻觉可能表现为虚构的事实、错误的引用、不存在的API或函数等。

## 幻觉的分类

### 事实性幻觉（Factuality Hallucination）
- **事实错误**：声称错误的事实信息（如"巴黎是英国的首都"）
- **实体错误**：虚构不存在的人物、地点、事件
- **数值错误**：给出错误的数字、日期、统计数据

### 忠实性幻觉（Faithfulness Hallucination）
- **指令违背**：回答偏离了用户的原始问题
- **上下文矛盾**：生成的回答与给定的上下文信息不一致
- **逻辑不一致**：回答前后矛盾或推理链条断裂

## 幻觉的成因

### 数据层面
- 训练数据包含错误信息或偏见
- 训练数据中的矛盾信息
- 某一领域训练数据不足
- 数据分布的长尾效应

### 模型层面
- 注意力机制可能关注错误的Token
- 解码策略（如Top-p采样、高temperature）引入随机性
- 最大似然训练目标过于简单，鼓励"流畅"而非"正确"
- 模型缺乏对"不知道自己不知道什么"的认知

### 推理层面
- 知识边界模糊：模型难以判断自己的知识边界
- 推理链过长导致错误累积
- 缺少外部验证机制

## 缓解方法

### RAG（检索增强生成）
将外部知识库检索结果作为回答的依据：
```
用户问题 → 检索相关文档 → 将文档作为上下文 → 生成基于文档的回答
```
RAG是目前最有效的幻觉缓解方法之一，使模型回答有据可查。

### Prompt层面对策
- 明确要求模型"如果不知道就说不知道"
- 要求模型引用信息来源
- 使用CoT引导分步推理
- 要求模型标注回答的置信度

### 解码策略调整
- 降低temperature（更确定性的输出）
- 使用事实性更强的采样策略
- 对关键事实进行后验证

### 模型训练对对策
- RLHF训练减少幻觉倾向
- 事实一致性作为训练目标
- 抗幻觉数据增强

### 后处理验证
- 使用NLI模型检查回答与来源的一致性
- 知识图谱验证事实三元组
- 搜索引擎交叉验证关键事实

### FactVerification
- 提取回答中的事实声明
- 对每个声明在知识库或搜索引擎中独立验证
- 标记未验证的声明

## 评估框架

### HaluEval
包含多种幻觉类型的评估数据集，用于衡量模型的幻觉倾向。

### TruthfulQA
测试模型在常识性、易误解问题上是否产生错误回答。

### FActScore
将模型回答分解为原子事实，然后在知识源中逐个验证每个事实的正确性。

### RAGAS
RAG系统的评估框架，包含忠实度（Faithfulness）、答案相关性、上下文相关性等指标。

## 未来方向

- **不确定性量化**：让模型输出置信度估计
- **事实核查集成**：持续监控和纠正输出
- **知识边界感知**：更好的"不知道自己不知道"能力
- **多源交叉验证**：结合多个知识源验证事实
