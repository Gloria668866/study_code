# 大语言模型训练流程

## 训练阶段概览

大语言模型（LLM）的训练通常分为多个阶段，从海量无标注数据的预训练开始，逐步增加人工标注和偏好信号，最终使模型能够遵循人类指令、提供有用且安全的回答。

## 第一阶段：预训练（Pre-training）

预训练是大模型训练的核心阶段，在海量文本语料上进行无监督/自监督学习。

### 数据准备
训练数据规模通常在TB到PB级别，数据来源包括：
- **网页数据**：Common Crawl、C4等公开网页数据集
- **书籍**：各类公开书籍和电子书
- **学术论文**：arXiv等公开论文库
- **代码**：GitHub等代码仓库
- **维基百科**：高质量的结构化知识
- **社交媒体和论坛数据**：Reddit、知乎等

### 数据预处理流程
```
原始数据 → 语言过滤 → 质量评分 → 去重（文档级+句子级）→ 
毒性过滤 → 个人身份信息（PII）去除 → 格式标准化 → 训练数据
```

### 训练目标
- **自回归语言模型（GPT式）**：给定前文预测下一个Token
  ```
  L = -Σ log P(x_t | x_1, ..., x_{t-1})
  ```
- **掩码语言模型（BERT式）**：随机掩码部分Token，预测被掩码的Token
- **前缀语言模型**：随机选择一个切分点，对前缀做双向注意力，对后缀做自回归生成

### 训练基础设施
- **模型并行**：将模型参数分片到多个GPU
- **数据并行**：每个GPU持有完整模型副本，处理不同数据分片
- **流水线并行**：将模型层分段部署在不同GPU上
- **ZeRO优化**：通过优化器状态、梯度、参数的分片减少显存占用
- **3D并行**：结合上述三种并行的混合策略
- **混合精度训练**：使用FP16/BF16减少显存和通信开销

## 第二阶段：监督微调（SFT）

### 目的
使用高质量的人工标注指令数据，让模型学会遵循人类指令的格式和风格。

### 数据格式
```json
{
  "instruction": "请解释什么是深度学习",
  "input": "",
  "output": "深度学习是机器学习的一个分支，基于人工神经网络..."
}
```

### 关键要素
- 数据质量远比数量重要，几千条高质量指令即可显著改善表现
- 需要覆盖多样化的任务类型：问答、摘要、代码、翻译、推理等
- 格式一致性有助于模型学习稳定的输出模式

## 第三阶段：人类反馈强化学习（RLHF）

### 三个步骤
1. **收集偏好数据**：对同一个提示的多个模型输出进行人类偏好排序
2. **训练奖励模型（Reward Model）**：用偏好数据训练一个能预测人类偏好的奖励模型
3. **PPO强化学习**：使用奖励模型作为奖励信号，用PPO算法优化LLM

### DPO（Direct Preference Optimization，直接偏好优化）
2023年提出的新方法，直接从偏好数据优化模型，无需单独训练奖励模型：
- 将RLHF的目标重新参数化，直接从偏好对中学习
- 简化了训练流程，避免了训练和调试独立奖励模型的复杂性
- 在多个基准测试中表现与PPO相当或更好

## 训练中的关键技术

### Tokenizer（分词器）
- **BPE（Byte-Pair Encoding）**：GPT系列使用，从字符开始逐步合并高频字符对
- **WordPiece**：BERT使用，基于似然的合并策略
- **SentencePiece**：LLaMA使用，将空格编码为显式字符，支持多种语言
- 分词器决定模型的词表大小和Token效率

### 学习率调度
- **Warmup**：训练初期逐步增加学习率，帮助稳定训练初期阶段
- **Cosine Decay**：余弦衰减是最常用的学习率衰减策略
- **Constant + Cooldown**：保持恒学习率，在训练末尾快速衰减

### 上下文窗口扩展
- RoPE位置编码具有良好的外推能力
- 通过插值或NTK-aware缩放等方法可以在不重新训练的情况下扩展上下文长度
- 更多扩展方法如YaRN、SelfExtend等，使模型支持百万Token级别的上下文

## 评估基准

- **MMLU**：多任务语言理解，涵盖57个学科
- **HumanEval**：代码生成能力评估
- **GSM8K**：数学推理能力
- **HellaSwag**：常识推理
- **TruthfulQA**：事实准确性
- **MT-Bench / AlpacaEval**：对话能力多轮评估
