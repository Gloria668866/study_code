# 提示工程（Prompt Engineering）

## 什么是提示工程

提示工程是设计和优化输入提示词以引导大语言模型生成期望输出的技术。好的提示能显著提升模型表现，无需修改模型参数，是使用大模型的基础技能。

## 核心原则

### 清晰明确
- 提供充分的上下文
- 指定输出格式（JSON、Markdown、列表等）
- 使用具体而非模糊的描述

### 结构化
- 使用分隔符划分不同部分（如```、###、---）
- 将复杂任务分解为步骤
- 提供输入和输出的明确边界

### 迭代优化
- 从简单提示开始，逐步增加复杂度
- 记录不同提示版本的效果
- 根据模型输出系统性地调整提示

## 常用技巧

### 角色设定（Persona）
为模型设定特定身份和专业领域，使其输出更符合期望风格：

```
你是一位有15年经验的Python高级工程师。
请审查以下代码，重点关注性能优化和安全隐患。
```

### 少样本提示（Few-shot Prompting）
在提示中提供几个示例，让模型通过类比学习：

```
将以下句子翻译为英文：
输入：今天天气很好 → 输出：The weather is very nice today
输入：我明天要去上班 → 输出：I will go to work tomorrow
输入：这本书很有趣 → 输出：
```

### 思维链（Chain-of-Thought, CoT）
引导模型逐步推理，显著提升复杂推理任务的准确率：

```
请一步步思考这个问题：
问题：小明有5个苹果，他吃了2个，又买了3个，然后给了小红1个。小明现在有多少个苹果？
思考过程：
1. 小明开始有5个苹果
2. 吃了2个后，还剩 5 - 2 = 3 个
3. 又买了3个，现在有 3 + 3 = 6 个
4. 给了小红1个，最终剩下 6 - 1 = 5 个
答案：5个苹果
```

### Zero-shot CoT
在没有示例的情况下，通过添加"让我们一步一步地思考"来触发模型的推理链。

### 自洽性（Self-Consistency）
对同一问题生成多条推理路径，取最一致的答案。结合CoT使用可进一步提升准确率。

### 思维树（Tree of Thoughts, ToT）
将问题解决建模为树搜索过程，在每个节点生成多个候选思路，评估每个分支，选择最有希望的路径继续探索。适合需要规划和探索的复杂问题。

### ReAct（Reasoning + Acting）
将推理和行动交替进行：模型在推理过程中可以调用外部工具（搜索、计算器等），根据工具返回结果继续推理。这是构建Agent的核心范式。

## 高级技巧

### 结构化输出
要求模型按特定格式输出，便于程序解析：
```
请以JSON格式输出，包含以下字段：
{
  "summary": "文章摘要",
  "keywords": ["关键词1", "关键词2"],
  "sentiment": "positive/negative/neutral"
}
```

### 负向提示
明确告诉模型不要做什么：
```
请不要给出过于技术性的回答，避免使用专业术语。
如果不知道答案，请直接说"我不知道"，不要猜测。
```

### 约束和边界
```
请用不超过100字总结以下内容。
请在回答中使用至少3个具体的例子。
请以初中生能理解的语言解释这个概念。
```

### 链式提示（Prompt Chaining）
将复杂任务分解为多个子任务，用上一个步骤的输出作为下一个步骤的输入：
1. 第一步：提取关键信息
2. 第二步：基于信息进行分析
3. 第三步：生成最终报告

### 自我反思（Self-Reflection）
让模型评估和修正自己的输出：
```
请先回答问题，然后检查你的回答是否存在错误或不一致之处，如果有请进行修正。
```

## Prompt工程的最佳实践

- 将指令放在提示的开头或结尾，使用###或"""等分隔符
- 在同一个提示中保持一致的格式和风格
- 对温度（temperature）等参数进行调优：创造性任务用较高的temperature，事实性任务用较低的temperature
- 系统地记录和比较不同提示的效果（版本化）
- 对于生产应用，建立Prompt的测试和评估体系

## 自动化Prompt工程

- **APE（Automatic Prompt Engineer）**：自动搜索最优提示词
- **DSPy**：将Prompt工程视为编程问题，通过编译器优化提示
- **Prompt Tuning**：在输入层添加可训练的软提示向量，通过梯度下降优化
