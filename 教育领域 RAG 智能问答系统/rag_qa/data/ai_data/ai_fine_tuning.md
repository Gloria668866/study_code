# 模型微调技术

## 微调 vs 预训练

预训练（Pre-training）在大规模通用语料上训练基础模型，学习语言的通用表示，耗时数周至数月，需要数千GPU。微调（Fine-tuning）在预训练模型基础上，使用特定领域的小规模标注数据继续训练，使模型适应特定任务，耗时数小时到数天。

## 全量微调（Full Fine-tuning）

更新模型的所有参数，效果最好但资源消耗最大。需要存储完整的模型副本和优化器状态。对于一个7B参数的模型，全量微调需要约56GB显存（仅模型权重）加上优化器状态和梯度共约200GB+。

## 参数高效微调（PEFT）

### LoRA（Low-Rank Adaptation，低秩适应）
核心思想：模型权重的更新矩阵具有低秩性，可以用两个小矩阵的乘积来近似。

```
W' = W + ΔW ≈ W + B × A
其中 B ∈ R^(d×r), A ∈ R^(r×k), r << min(d,k)
```

**优势**：
- 只训练额外添加的低秩矩阵（通常几百KB到几MB）
- 原始模型权重保持不变
- 可以针对不同任务训练多个LoRA模块，灵活切换
- 推理时可以将LoRA权重合并到原始权重中，不增加推理延迟

**关键参数**：
- **r（秩）**：低秩矩阵的秩，通常4-64，r越大表达能力越强
- **alpha（缩放因子）**：控制LoRA更新的幅度
- **target_modules**：应用LoRA的层，通常选择注意力层的Q、K、V、O投影矩阵

### QLoRA
Quantized LoRA，将基础模型量化为4-bit（NF4量化格式），然后在此之上训练LoRA适配器：
- 将显存需求降低4倍以上
- 7B模型在单张RTX 3090（24GB）上即可微调
- 效果接近全量微调
- 使用了双重量化（Double Quantization）和分页优化器（Paged Optimizers）

### Adapter
在Transformer层之间插入小型适配器模块，只训练这些新增参数：
- 每个Adapter包含下投影→激活函数→上投影
- 参数量约为原模型的0.5-8%
- 早期的参数高效微调方法

### Prefix Tuning / Prompt Tuning
在输入层添加可训练的虚拟Token（软提示）：
- Prefix Tuning：在每层添加可训练的前缀
- Prompt Tuning：仅在输入层添加软提示Token
- 参数量极少但效果相对有限

### IA3（Infused Adapter by Inhibiting and Amplifying Inner Activations）
通过可学习的缩放向量来调整注意力机制和FFN的激活值，参数效率极高。

## 指令微调（Instruction Tuning）

### 数据构建
- 收集多样化的任务指令数据
- 格式统一为 instruction-input-output 三元组
- 覆盖推理、生成、分类、代码等多种任务
- 数据质量远比数据量重要

### 训练技巧
- 只对输出部分计算损失（mask掉输入和提示部分）
- 使用较小的学习率（通常为预训练的1/10到1/100）
- 使用cosine学习率调度和warmup
- 控制训练轮数防止过拟合（通常1-3个epoch）

### 数据增强
- **Self-Instruct**：使用强模型自动生成指令数据
- **Evol-Instruct**：逐步增加指令的复杂度
- **Backtranslation**：从高质量输出反推指令

## 对齐微调

### RLHF（人类反馈强化学习）
通过人类偏好数据对齐模型行为与人类价值观：
1. SFT阶段：监督微调建立基本指令遵循能力
2. 奖励模型训练：基于人类偏好比较数据训练奖励模型
3. PPO训练：使用奖励模型优化策略

### DPO（直接偏好优化）
绕过独立的奖励模型，直接从偏好数据中优化模型：
```
L_DPO = -log σ(β(log π_θ(y_w|x)/π_ref(y_w|x) - log π_θ(y_l|x)/π_ref(y_l|x)))
```
其中y_w是优选回复，y_l是较差的回复，π_ref是参考模型。

### KTO、ORPO等新方法
持续涌现的对齐方法，减少对成对偏好数据的依赖，或者将SFT和对齐合并在一个阶段。

## 微调的评估

### 自动化评估
- 使用GPT-4等强模型作为裁判进行评估
- AlpacaEval、MT-Bench等标准化评估集
- 针对特定任务的自定义评估集

### 人工评估
- A/B测试：对比微调前后的输出
- 盲测排序：按质量排序多个模型的输出
- 领域专家评审：特定领域的专业评估

### 监控指标
- 训练损失和验证损失曲线
- 各任务的独立评估指标
- 安全性和拒绝率（不回答不当问题的比例）

## 实战建议

1. 从QLoRA开始，成本低且效果接近全量微调
2. 优先保证训练数据质量，删除低质量样本
3. 建立自动化评估集，快速迭代
4. 使用vLLM或TGI进行高效推理部署
5. 注意过拟合风险，监控验证集指标
