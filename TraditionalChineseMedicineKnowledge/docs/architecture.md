# 架构说明

## 目标

本项目将原始中医药知识图谱实验代码重构为可演示、可扩展的 AI 大模型应用：中医药知识图谱增强问答系统。

## 核心链路

```text
抽取 JSON 数据 -> 标准化文档/图谱记录 -> FAISS 索引 -> FastAPI 问答服务 -> Streamlit 前端
                                      └-> Neo4j 图谱导入 -> GraphRAG 增强
```

## 双模式

### Local RAG

当 Neo4j 未配置或不可用时，系统使用本地 FAISS 检索中医药资料，将召回资料作为上下文交给 DeepSeek 生成回答。

### GraphRAG

当 Neo4j 可用时，系统会优先根据向量召回实体查询相关图谱关系，将实体关系与文档资料共同传入大模型生成回答。

## 安全边界

系统定位为中医药知识查询和 AI 技术演示，不给出确定性诊断、处方或剂量建议。
